# HostelConnect - Complaint Prioritization System

A responsive, static prototype for a hostel complaint management system.

## How to Run
1.  Simply open `index.html` in any modern web browser (Edge, Chrome, Firefox).
2.  No server or database installation is required.

## Features
-   **Static Architecture**: HTML5, CSS3, JavaScript (ES6).
-   **Data Persistence**: Uses browser's `localStorage` to save complaints and profiles (data persists even if you close the browser, until cache is cleared).
-   **Themes**: Light and Dark mode support.

## Usage Flow
1.  **Log in as a Student**: Submit a complaint (e.g., "Fan broken", "High Urgency").
2.  **Log out**.
3.  **Log in as Admin**: Go to the dashboard. You will see the new complaint.
4.  **Update Status**: Change status from "Submitted" to "In Progress".
5.  **Log out and back in as Student**: check "My Complaints" to see the status update.
